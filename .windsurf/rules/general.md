# SafeMoto General Development Rules

## Code Standards
- Use JavaScript only - no TypeScript
- Follow ES6+ standards and modern JavaScript practices
- Use meaningful variable and function names
- Keep functions small and focused on single responsibility
- Add comments for complex logic and business rules

## React Guidelines
- Use functional components with hooks
- Prefer custom hooks for reusable logic
- Implement proper state management
- Use React.memo for performance optimization
- Follow React best practices for component structure

## File Organization
- Keep components in logical folders
- Use index files for clean imports
- Separate business logic from UI components
- Organize utilities and helpers properly
- Maintain consistent file naming conventions

## Performance Requirements
- Optimize for mobile devices and slow networks
- Implement lazy loading for heavy components
- Use code splitting for better performance
- Minimize bundle size and optimize assets
- Test on various mobile devices and network conditions

## Quality Assurance
- All code must be reviewed before merging
- Write tests for critical functionality
- Use ESLint and maintain code quality
- Test accessibility features
- Ensure cross-browser compatibility

## Git Workflow
- Use feature branches for new development
- Write clear, descriptive commit messages
- Keep PRs focused and manageable
- Ensure all tests pass before merging
- Document any breaking changes
